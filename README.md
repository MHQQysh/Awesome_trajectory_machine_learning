# awesome_trajectory_machine_learning

# Model Trajectory Papers

Last updated: 2026/07/03

This list follows the table style of [yaolinli/MLLM-Token-Compression](https://github.com/yaolinli/MLLM-Token-Compression), but the scope is **model trajectory** rather than token compression.

Scope used here:
- **Weight / checkpoint trajectory**: model parameters, expert checkpoints, training paths, loss-basin paths, model fusion paths.
- **Hidden-state / representation trajectory**: layer-wise or decoding-time hidden states, reasoning paths, latent thought trajectories.
- **Prediction trajectory**: vocabulary/logit distribution evolution across layers or steps.
- **OT / flow / diffusion trajectory**: optimal transport, Sinkhorn, Neural ODE/CDE, flow matching, diffusion over weights or hidden states.

Excluded by default: physical trajectory prediction for pedestrians, vehicles, robotics, and autonomous driving, unless the paper explicitly studies model internals or model parameters.

## Quick Reading Map

- **If you mean hidden-state trajectories inside LLMs**: start with `LLM Reasoning as Trajectories`, `Truth as a Trajectory`, `TrajGuard`, `Lines of Thought`, and `Trajectory Dynamics in Language Model Hidden States`.
- **If you mean predicting or generating model weights**: start with `Neural Network Diffusion`, `Recurrent Diffusion for Large-Scale Parameter Generation`, `Efficient Training with Denoised Neural Weights`, `DeepWeightFlow`, and `WeightFlow`.
- **If you mean OT / transport paths between models**: start with `Model Fusion via Optimal Transport`, `Wasserstein Barycenter-based Model Fusion`, `Git Re-Basin`, `Re-basin via Implicit Sinkhorn Differentiation`, and `OT-Transformer`.
- **If you mean training trajectory as supervision**: start with `Dataset Distillation by Matching Training Trajectories`, `DATM`, and `Sharpness-Aware Trajectory Matching`.
