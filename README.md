## Tag Description

- ![Venue](https://img.shields.io/badge/Venue-blue) conference, journal, workshop, technical report, or arXiv source.
- ![Object](https://img.shields.io/badge/Object-purple) trajectory object or data type, such as weight, hidden state, prediction, reasoning, dataset, model, graph, or manifold.
- ![Position](https://img.shields.io/badge/Position-cyan) where the trajectory or geometry lives, such as training, layer, decoding, latent space, weight space, ODE, or fusion.
- ![Method](https://img.shields.io/badge/Method-lightgrey) method or topic tag, such as OT, Sinkhorn, diffusion, flow, lens, re-basin, distillation, steering, or dynamics.
- ![Usage](https://img.shields.io/badge/Usage-yellow) intended use case, such as analyze, predict, generate, control, fuse, defense, compress, optimize, or embed.
- ![Stage](https://img.shields.io/badge/Stage-orange) model lifecycle stage: train, infer, or eval.
- ![Relation](https://img.shields.io/badge/Relation-yellowgreen) relevance to model trajectory: core, related, or background.

## Quick Reading Map

- **If you mean hidden-state trajectories inside LLMs**: start with `LLM Reasoning as Trajectories`, `Truth as a Trajectory`, `TrajGuard`, `Lines of Thought`, and `Trajectory Dynamics in Language Model Hidden States`.
- **If you mean layer-wise geometry of transformer representations**: start with `Trajectory Geometry of Transformer Representations Across Layers`, `Transformers Represent Belief State Geometry in their Residual Stream`, `Lines of Thought`, and `Latent Trajectory Dynamics in LLMs`.
- **If you mean visual information flow in MLLMs / LVLMs**: start with `HiMAP`, `FlowCut`, `VFlowOpt`, and `From Redundancy to Relevance: Information Flow in LVLMs Across Reasoning Tasks`.
- **If you mean layer-wise prediction or answer-logit trajectories**: start with `Tuned Lens`, `LogitLens4LLMs`, and `Transformer Feed-Forward Layers Build Predictions by Promoting Concepts in the Vocabulary Space`.
- **If you mean progressive token pruning as a decision trajectory**: start with `TPRL`, `TOP-RL`, and `Reinforcement Learning-based Token Pruning in Vision Transformers`.
- **If you mean predicting or generating model weights**: start with `Neural Network Diffusion`, `Recurrent Diffusion for Large-Scale Parameter Generation`, `Efficient Training with Denoised Neural Weights`, `DeepWeightFlow`, and `WeightFlow`.
- **If you mean OT / transport paths between models**: start with `Model Fusion via Optimal Transport`, `Wasserstein Barycenter-based Model Fusion`, `Git Re-Basin`, `Re-basin via Implicit Sinkhorn Differentiation`, and `OT-Transformer`.
- **If you mean training trajectory as supervision**: start with `Dataset Distillation by Matching Training Trajectories`, `DATM`, and `Sharpness-Aware Trajectory Matching`.
